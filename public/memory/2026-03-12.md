### OceanicLog Automation Architecture
- iPad GPS (CoreLocation) tracks real-time position — knows exact second fix is crossed
- Auto-notification: "Approaching 52N040W in 2 min" → pilot confirms
- Time logged automatically at crossing moment (no manual entry)
- Fuel = only required pilot input (one number, ~15 seconds)
- Altitude = pre-filled from flight plan, pilot corrects if stepped
- Apple Shortcuts: auto-fires when pilot takes FMS screenshot → AI extracts fuel/altitude → pre-fills form → pilot taps confirm
- Wind data: cross-reference live upper-air data at fix position automatically
- Net result: ~15 seconds per fix, nearly zero effort — nothing in market does this
- Key insight: GPS handles WHEN + WHERE, AI handles WHAT (fuel/altitude from screenshot), pilot just confirms
- iOS background location ("Always" permission) works screen-locked — like ForeFlight
- Battery note: needs iPad plugged in for long crossings (standard in cockpit anyway)

### OceanicLog Full Product Spec (defined March 12)
**Preflight:** flight plan upload → auto-parse route/ETPs/fuel/times; crew names/certs; SELCAL; OCA clearance; block fuel
**In-flight:** GPS auto-captures position + time at each fix; pilot enters fuel (15 sec); app logs GNE check, SELCAL log, deviations
**Post-flight:** pilot taps "COMPLETE" → enters landing fuel + remarks → PDF generated

**PDF structure (4 pages):**
1. Flight Summary (planned vs actual: times, fuel, distance, performance)
2. Oceanic Position Reports (full table + track chart)
3. Fuel Analysis (segment burns, chart, planned vs actual)
4. Crew & Compliance (certs, SELCAL log, OCA clearance, MNPS/RVSM statement, audit signature)
