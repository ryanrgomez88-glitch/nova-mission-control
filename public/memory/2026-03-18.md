## Email Poll Spam Fix — Commit 74e38e6 (6:45 PM CDT)

Ryan reported Otto was still checking emails too frequently.

### Root Cause
Every git push triggers a Railway redeploy + restart, which fired a Gmail check 10 seconds after boot. ~7 commits pushed today = ~7 email notifications.

### Fix (commit 74e38e6)
1. **30-min cooldown guard** — `lastPollAt` timestamp + `POLL_COOLDOWN = 30 * 60 * 1000` — skips check if ran within 30 min
2. **Startup delay: 10s → 5 min** — `setTimeout(checkGmailForInvoices, 5 * 60 * 1000)` gives Railway time to stabilize
3. **12-hour interval unchanged** for normal operation

### Key Lesson
Future deploys won't trigger email spam. Cooldown is in-memory so resets on restart, but the 5-min delay + 30-min cooldown means even rapid sequential deploys stay quiet.
