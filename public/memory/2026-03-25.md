## PRODUCT PRINCIPLE — Fuzzy Category Matching (Ryan, ~8:26 PM)
Ryan's core feedback: OpsAI must NOT crash or break when users use non-exact terminology.
"Airline" vs "Airline Expense" vs "Travel" should all resolve to the same category.

### The pattern:
- Maintain a **per-user category alias table** in Supabase: `user_id + raw_input → category_id`
- Fuzzy/semantic match user input against known categories
- High confidence → resolve silently
- Ambiguous → confirm once: *"I'm mapping 'airline' → Airline, correct?"*
- Store the alias — never ask again for that user
- Applies to: categories, vendor names, crew names, airport codes, trip identifiers
- Core principle: **humans don't talk in database keys**
